# 欺上瞒下 - 双重伪装策略

## 架构背景
- **中转机**：CN2 GIA 线路（优质线路），IP 风控值 ~30（中等质量，必须保护）
- **落地机**：普通线路，IP 质量好（干净的美国 IP）

## 核心原则：欺上瞒下

### 欺上（欺骗 GFW）
**让 GFW 把中转机看成"小透明"，完全不可疑**
- 中转机像普通 CDN，没有代理特征
- Layer 4 盲传，不解密，不检查
- 错误 SNI → Apple CDN（完全沉默）
- 流量模式正常，没有异常

### 瞒下（欺骗目标网站）
**让目标网站把用户看成"正常美国本地人"**
- 美国 IP + 美国指纹
- 没有异常流量、没有异常行为
- 享受真正美国人的待遇
- 是个良民，完全没有风险

## 技术实现

### 欺上（对 GFW）- 中转机隐蔽性（P0 最高优先级）

**核心目标：保护中转机不被 GFW 封 IP**

中转机直接面对 GFW，是最容易被探测和封禁的节点。必须确保中转机在 GFW 面前完全隐蔽。

#### 关键需求
- [ ] **错误 SNI 必须路由到 Apple CDN**（完全沉默，不返回任何响应）
- [ ] **不能有任何异常响应**（不能泄露代理/Xray 特征）
- [ ] **流量特征必须正常**（Layer 4 盲传，不解密，不检查）
- [ ] **端口行为必须像 CDN**（443 端口，TLS SNI 路由）
- [ ] **不能有任何主动探测特征**（超时、RST、特殊响应等）
- [ ] **日志不能泄露敏感信息**（不记录 SNI、目标 IP 等）

#### 配置检查清单
1. ✅ 是否使用 SNI 路由（Layer 4 盲传）？
2. ✅ 错误 SNI 是否路由到 Apple CDN（完全沉默）？
3. ✅ 是否有任何响应泄露中转机身份？
4. ✅ 配置文件中是否使用域名而非硬编码 IP？
5. ✅ 日志是否关闭或不记录敏感信息？
6. ✅ 防火墙规则是否正确（只开放 443 和受限 SSH）？
7. ✅ 是否有任何主动探测特征（超时、RST 等）？

### 瞒下（对目标网站）- 落地机美国人伪装（P1 次要优先级）

**核心目标：让目标网站把用户看成正常美国本地人**

落地机有中转机保护，但仍需伪装成美国本地用户，避免被目标网站识别为代理。

#### 关键需求
- [ ] **美国 IP**（干净的美国 IP，没有代理历史）
- [ ] **美国指纹**（TLS 指纹、HTTP 头、时区、DNS 等）
- [ ] **正常流量模式**（不是机器人，像真实用户）
- [ ] **标准 TLS 1.3**（不使用 REALITY 等特殊协议）
- [ ] **真实网站 fallback**（主动探测时返回真实网站内容）
- [ ] **防火墙白名单**（只允许中转机 IP）

#### 配置检查清单
1. Xray inbound 是否有 `rejectUnknownSni: true`？
2. Nginx fallback 是否返回真实网站内容？
3. TLS 配置是否使用标准协议（不使用 REALITY）？
4. 防火墙是否只允许中转机 IP？
5. 是否有任何日志或错误消息泄露代理信息？
6. 出口流量是否有美国指纹（时区、DNS、HTTP 头等）？

### 连接链路保护（P1）
- [ ] 中转机不能在配置中硬编码落地机 IP（使用域名解析）
- [ ] 中转机到落地机的连接必须加密（TLS）
- [ ] 错误 SNI 不能泄露落地机信息

## 检查清单

### 连接测试
1. 从外部主动探测中转机，使用错误 SNI，是否完全沉默？
2. 从外部主动探测落地机，是否返回正常网站？
3. 落地机是否拒绝非中转机 IP 的连接？
4. TLS 握手是否正常（无异常指纹）？
5. 出口流量是否像美国本地用户？

## 风险等级
- **高风险**：中转机被 GFW 识别 → 立即被封（P0 保护目标）
- **中风险**：落地机被目标网站识别为代理 → 账号被封、服务受限
- **低风险**：连接不稳定 → 用户体验差

## 优先级
1. **P0**：中转机隐蔽性（保护不被 GFW 封 IP）
2. **P1**：落地机美国人伪装（保护不被目标网站识别）
3. **P2**：连接稳定性和性能
