# v5.20 版本发布说明

## 📦 版本信息
- **版本号**: v5.20
- **发布日期**: 2026-05-02
- **基础版本**: v5.19
- **修复类型**: 精准手术式修复（基于三份审查报告）

## 🎯 修复目标
根据三份独立审查报告的建议，本版本专注于修复v5.19中的关键问题，同时保持架构稳定性。

## 🔴 严重问题修复（CRITICAL）

### 1. [落地机] 删除假证书延迟逻辑
**问题描述**: v5.19的`issue_certificate()`函数中生成了后台脚本但从未执行，纯属废代码  
**审查报告**: 审查者AI指出"这段代码仅仅把sleep 30m写进了一个文本里，然后主脚本根本没有等待"  
**修复方案**: 完全删除延迟逻辑，直接申请证书（Let's Encrypt的速率限制针对大批量申请，单机部署无需延迟）  
**影响**: 简化代码，消除混淆，加快安装速度

### 2. [中转机] 恢复IPv6动态探测
**问题描述**: v5.19的`have_ipv6()`函数硬编码`return 1`，导致所有ip6tables代码变成死代码  
**审查报告**: 审查者AI指出"强行return 1看似没问题，但导致脚本下面成百上千行的ip6tables防火墙逻辑全部变成了死代码"  
**修复方案**: 恢复完整的IPv6探测逻辑，但默认禁用（可通过环境变量`TRANSIT_FORCE_IPV6=1`启用）  
**影响**: 保持未来扩展性，避免代码腐化

### 3. [落地机] 澄清IPv6双栈策略
**问题描述**: v5.19代码逻辑正确（仅纯IPv4禁用IPv6），但注释误导  
**审查报告**: 审查者AI指出"v5.19实际上已正确实现双栈保留"  
**修复方案**: 更新注释，明确说明策略  
**影响**: 消除误解，代码逻辑无变化

## 🟠 重要优化（HIGH）

### 4. [落地机] 添加轻量级流量伪装
**问题描述**: v5.19完全删除流量模拟后，VPS只有代理流量，特征单一  
**审查报告**: 审查者AI指出"完全删除流量模拟后，VPS只有代理流量，没有正常用户的背景流量，这会让流量特征过于单一"  
**修复方案**: 
- 新增`setup_background_traffic()`函数
- 每天随机时间（2个时间点）访问2-4个常见网站（google.com, youtube.com等）
- 使用真实浏览器User-Agent
- 随机间隔30-120秒，模拟真实浏览行为
**影响**: 增强"欺上瞒下"效果，符合用户核心诉求

### 5. [落地机] 完善1Panel端口持久化
**问题描述**: EXTRA_PORTS未写入防火墙开机自启脚本，重启后1Panel断网  
**审查报告**: 审查者AI指出"额外放行端口必须写入防火墙的开机自启脚本(firewall-restore.sh)中"  
**修复方案**: 在`_persist_iptables()`函数中读取EXTRA_PORTS并写入防火墙恢复脚本  
**影响**: 重启后1Panel不会断网，完美兼容

### 6. [中转机] 添加CN2 GIA优化
**问题描述**: 未针对CN2 GIA线路优化TCP参数  
**审查报告**: 审查者AI建议"添加TCP优化：tcp_slow_start_after_idle=0等"  
**修复方案**: 在`optimize_kernel_network()`中添加CN2 GIA专用参数  
**影响**: 提升CN2线路性能

## 🟡 次要改进（MEDIUM）
